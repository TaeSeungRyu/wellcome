## Diff 알고리즘 상세 설명

Virtual DOM에서 **Diff 알고리즘**은
이전 Virtual DOM 트리와 새로운 Virtual DOM 트리를 비교하여
**변경된 부분만 실제 DOM에 반영**하기 위한 핵심 알고리즘입니다.

대표적으로 **React**에서 사용하는 Reconciliation 과정이 이에 해당합니다.

---

## 1️⃣ Diff 알고리즘이 필요한 이유

DOM 트리를 완전 탐색하여 비교하면 시간 복잡도는:

```
O(n³)
```

* 노드 재배치
* 노드 추가/삭제
* 속성 변경

대규모 UI에서는 매우 비효율적입니다.

👉 React는 **현실적인 가정(Heuristic)**을 기반으로
이를 **O(n)** 수준으로 최적화했습니다.

---

## 2️⃣ React의 Diff 알고리즘 가정 (Heuristic)

### 📌 가정 1: 서로 다른 타입의 노드는 다른 트리다

```jsx
<div> → <span>
```

* 타입이 다르면 기존 트리 제거
* 새로 생성

👉 전체 subtree 교체

---

### 📌 가정 2: 같은 타입이면 속성만 비교

```jsx
<div class="red"> → <div class="blue">
```

* DOM 노드 유지
* 변경된 attribute만 수정

---

### 📌 가정 3: 리스트는 key를 기준으로 비교

```jsx
<li key="1">A</li>
<li key="2">B</li>
```

`key`를 통해 노드를 식별

* key가 같으면 재사용
* key가 다르면 교체

👉 key가 없으면 인덱스 기반 비교 → 성능 저하 가능

---

## 3️⃣ Diff 알고리즘 단계별 동작

### 1단계: 루트 노드 비교

* 타입이 다르면 전체 교체
* 같으면 속성 비교

---

### 2단계: 자식 노드 비교

기본 전략:

* 동일 위치의 노드끼리 비교
* 순차 탐색

예시:

```
A B C
A C B
```

👉 key 없으면 B,C 모두 변경으로 인식
👉 key 있으면 위치만 이동 처리

---

### 3단계: 변경 사항 패치(Patch) 생성

* Update
* Replace
* Remove
* Insert

변경 목록을 만든 후 실제 DOM에 한 번에 적용

---

## 4️⃣ 시간 복잡도

| 방식         | 시간 복잡도 |
| ---------- | ------ |
| 일반 트리 비교   | O(n³)  |
| React Diff | O(n)   |

👉 핵심은 **완전 탐색을 하지 않는 것**

---

## 5️⃣ 리스트 비교 예시

### ❌ key 없음

```jsx
[A, B, C] → [C, A, B]
```

* 모두 변경으로 인식 가능

---

### ✅ key 있음

```jsx
[{id:1}, {id:2}, {id:3}]
```

* key 기준 매칭
* 실제 이동만 처리

👉 성능 최적화

---

## 6️⃣ 실제 DOM 반영 과정 (Reconciliation)

1. 새로운 Virtual DOM 생성
2. 이전 Virtual DOM과 비교 (Diff)
3. 변경 사항 계산
4. 실제 DOM 최소 변경 적용

---

## 7️⃣ 핵심 정리

* Diff 알고리즘은 Virtual DOM의 핵심
* 타입이 다르면 subtree 교체
* 타입이 같으면 속성 비교
* 리스트는 key 기반 비교
* 시간 복잡도 O(n) 최적화
* 변경된 부분만 실제 DOM에 반영

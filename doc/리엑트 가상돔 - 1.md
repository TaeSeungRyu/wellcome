## Virtual DOM 이론 정리

### 1️⃣ Virtual DOM이란?

**Virtual DOM(가상 DOM)**은 실제 브라우저의 **DOM(Document Object Model)**을 직접 수정하지 않고,
**메모리 상에 가상의 DOM을 만들어 변경 사항을 먼저 적용한 뒤**,
실제 DOM에는 **필요한 부분만 최소한으로 반영하는 방식**입니다.

주로 **React**와 같은 라이브러리에서 사용하는 핵심 개념입니다.

---

### 2️⃣ 왜 필요한가?

브라우저의 실제 DOM 조작은 비용이 큽니다.

* DOM 수정 → Reflow → Repaint 발생
* 잦은 변경 시 성능 저하
* 대규모 UI에서 비효율적

👉 따라서 **실제 DOM 변경을 최소화**하는 것이 중요합니다.

---

### 3️⃣ 동작 원리

Virtual DOM은 다음과 같은 과정을 거칩니다:

#### 1. 초기 렌더링

* Virtual DOM 생성
* 실제 DOM에 반영

#### 2. 상태(State) 변경 발생

* 새로운 Virtual DOM 생성

#### 3. Diff 알고리즘 실행

* 이전 Virtual DOM과 새로운 Virtual DOM 비교
* 변경된 부분만 탐색

#### 4. 실제 DOM에 최소 변경 적용 (Reconciliation)

* 필요한 부분만 업데이트

---

### 4️⃣ Diff 알고리즘 기반 비교

Virtual DOM의 핵심은 **Diff 알고리즘**입니다.

비교 전략:

1. **같은 타입의 노드는 재사용**
2. **다른 타입이면 전체 교체**
3. 리스트 비교 시 `key` 값 기준으로 비교
4. 변경된 부분만 실제 DOM에 반영

이를 통해:

* 전체 DOM을 다시 그리지 않음
* 성능 최적화 가능

---

### 5️⃣ 실제 DOM vs Virtual DOM 비교

| 구분    | 실제 DOM            | Virtual DOM |
| ----- | ----------------- | ----------- |
| 위치    | 브라우저              | 메모리         |
| 속도    | 느림                | 빠름          |
| 변경 방식 | 직접 수정             | 비교 후 최소 반영  |
| 비용    | Reflow/Repaint 발생 | 최소화         |

---

### 6️⃣ 핵심 요약

* Virtual DOM은 **실제 DOM 변경을 최소화**하기 위한 기술
* **Diff 알고리즘 기반 비교**
* 변경된 부분만 실제 DOM에 반영
* 대규모 UI에서 성능 향상

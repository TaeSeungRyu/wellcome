## React Reconciliation 이론 정리

**Reconciliation(재조정)**은
이전 Virtual DOM과 새로운 Virtual DOM을 비교(Diff)하여
**변경된 부분만 실제 DOM에 반영하는 과정**입니다.

대표적으로 **React**의 핵심 렌더링 최적화 메커니즘입니다.

---

## 1️⃣ 왜 필요한가?

React는 상태(State)가 변경될 때마다:

1. 새로운 Virtual DOM 생성
2. 이전 Virtual DOM과 비교
3. 변경된 부분만 실제 DOM에 적용

👉 매번 전체 DOM을 다시 그리는 것이 아니라
**최소한의 DOM 조작만 수행**합니다.

---

## 2️⃣ Reconciliation 동작 과정

### ① 상태 변경 발생

* `setState` 또는 `useState` 호출

### ② 새로운 Virtual DOM 생성

* 변경된 UI 구조가 메모리에 생성됨

### ③ Diff 알고리즘 실행

이전 트리와 새로운 트리를 비교

### ④ 실제 DOM 업데이트

* 변경된 부분만 patch 적용

---

## 3️⃣ 핵심 비교 규칙 (Heuristic)

### 1. 타입이 다르면 전체 교체

```jsx
<div /> → <span />
```

* 기존 노드 제거
* 새로운 노드 생성

---

### 2. 타입이 같으면 속성만 비교

```jsx
<div className="red" />
→
<div className="blue" />
```

* DOM 노드 유지
* class만 수정

---

### 3. 리스트는 key 기준 비교

```jsx
<li key="1">A</li>
<li key="2">B</li>
```

* key가 같으면 재사용
* key가 다르면 새로 생성

👉 key는 Reconciliation 성능 최적화의 핵심

---

## 4️⃣ Reconciliation의 특징

* 전체 트리를 완전 탐색하지 않음
* O(n³) 대신 O(n) 비교
* 변경 사항을 batch 처리
* 실제 DOM 조작 최소화

---

## 5️⃣ Diff vs Reconciliation 차이

| 구분 | Diff           | Reconciliation  |
| -- | -------------- | --------------- |
| 역할 | Virtual DOM 비교 | 실제 DOM 반영 과정 전체 |
| 범위 | 비교 알고리즘        | 비교 + 패치 적용      |

👉 **Diff는 Reconciliation의 일부 과정**

---

## 6️⃣ 한 줄 요약

Reconciliation은
**Virtual DOM을 비교하여 실제 DOM을 최소한으로 업데이트하는 React의 핵심 렌더링 전략**입니다.

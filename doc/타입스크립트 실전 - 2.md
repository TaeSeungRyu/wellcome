
# 🔥 11. API 응답 래핑 패턴 (실무 필수)

실제 API는 이렇게 안 깔끔하게 옴 👇

```ts
type ApiResponse<T> = {
  data: T;
  message: string;
  success: boolean;
};

async function getUser(): Promise<ApiResponse<User>> {
  const res = await fetch("/api/user");
  return res.json();
}
```

👉 핵심

* **제네릭 + API 구조 통일**
* axios / react-query에서 매우 자주 씀

# 🔥 12. Nullable vs Optional 구분 (중요)

```ts
type User = {
  name?: string;        // 없을 수도 있음 (undefined)
  email: string | null; // 값은 있지만 null 가능
};
```

👉 핵심

* `?` = 값 자체가 없음
* `null` = 값은 있지만 비어있음
  👉 API 설계 이해에 중요 (백엔드랑 충돌 많이 나는 지점)

# 🔥 13. Discriminated Union (상태 설계 핵심)

```ts
type State =
  | { status: "loading" }
  | { status: "success"; data: User }
  | { status: "error"; error: string };

function render(state: State) {
  if (state.status === "success") {
    console.log(state.data);
  }
}
```

👉 핵심

* **if문에서 자동으로 타입 좁혀짐**
* React 상태 관리에서 매우 강력함

# 🔥 14. keyof + typeof 조합 (진짜 많이 씀)

```ts
const user = {
  id: 1,
  name: "kim",
};

type UserKeys = keyof typeof user; // "id" | "name"
```

👉 핵심

* 객체 기반으로 타입 자동 생성
* 하드코딩 줄이는 핵심 기술

# 🔥 15. 인덱스 시그니처 vs Record 차이

```ts
// 느슨함
type AnyMap = {
  [key: string]: string;
};

// 엄격함
type Role = "admin" | "user";
type RoleMap = Record<Role, string>;
```

👉 핵심

* Record가 더 안전
* 인덱스 시그니처는 최소화

# 🔥 16. 함수 오버로드 (고급 패턴)

```ts
function format(value: string): string;
function format(value: number): number;
function format(value: string | number) {
  return value;
}
```

👉 핵심

* 입력에 따라 반환 타입 다르게
* 라이브러리 구현에서 많이 씀

# 🔥 17. unknown vs any (중요도 높음)

```ts
let a: any = 10;
let b: unknown = 10;

// any → 바로 사용 가능 (위험)
// unknown → 검사 후 사용
```

👉 핵심

* 실무에서는 **unknown → 타입가드 조합이 정석**

# 🔥 18. Deep 타입 (중첩 구조 대응)

```ts
type DeepPartial<T> = {
  [K in keyof T]?: DeepPartial<T[K]>;
};
```

👉 핵심

* 폼 수정 / 설정 객체에서 자주 씀
* 라이브러리에서 많이 등장

# 🔥 19. satisfies (TS 최신 핵심)

```ts
const user = {
  name: "kim",
  age: 20,
} satisfies { name: string; age: number };
```

👉 핵심

* 타입 체크는 하되, **추론은 유지**
* as 보다 안전

# 🔥 20. 브랜딩 타입 (고급, 실무 고수 느낌)

```ts
type UserId = number & { readonly brand: unique symbol };

function getUser(id: UserId) {}
```

👉 핵심

* 같은 number라도 의미 구분
* 큰 서비스에서 버그 방지용

# 💡 진짜 실무 감각 요약

여기서 중요한 건 “문법”이 아니라 패턴이야:

👉 초급

* 타입 붙이기
* Partial / Union

👉 중급

* Discriminated Union
* Generic + API 구조화
* keyof / typeof

👉 고급

* satisfies
* 브랜딩
* 타입 설계 (상태 모델링)

# 🚀 현실적인 조언

솔직히 말하면 👇

* 문법 20개 아는 사람보다
* **상태를 타입으로 잘 설계하는 사람이 훨씬 잘함**

예를 들면:

```ts
// ❌ 나쁜 설계
type State = {
  loading: boolean;
  data?: User;
  error?: string;
};

// ✅ 좋은 설계
type State =
  | { status: "loading" }
  | { status: "success"; data: User }
  | { status: "error"; error: string };
```

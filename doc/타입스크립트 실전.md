# 🔥 1. API 응답 타입 정의

백엔드에서 오는 데이터는 항상 타입을 정의합니다.

```ts
type User = {
  id: number;
  name: string;
  email: string;
};

async function getUser(): Promise<User> {
  const res = await fetch("/api/user");
  return res.json();
}
```

👉 핵심:

* **Promise<T> 형태 기억**
* API 응답은 무조건 타입 붙이기

---

# 🔥 2. Partial / Pick / Omit (유틸 타입 3대장)

기존 타입을 “조합”해서 씀

```ts
type User = {
  id: number;
  name: string;
  email: string;
};

// 일부만 수정할 때
type UpdateUser = Partial<User>;

// 일부만 가져올 때
type UserName = Pick<User, "name">;

// 일부 제거할 때
type UserWithoutEmail = Omit<User, "email">;
```

👉 핵심:

* CRUD 만들 때 거의 필수

---

# 🔥 3. Union + Literal 타입 (상태 관리)

상태를 안전하게 관리할 때

```ts
type Status = "loading" | "success" | "error";

function handle(status: Status) {
  if (status === "loading") {
    console.log("로딩중");
  }
}
```

👉 핵심:

* 문자열 대신 **정해진 값만 허용**
* 버그 방지 효과 큼

---

# 🔥 4. 타입 가드 (Type Guard)

실무에서 엄청 많이 씀

```ts
function isString(value: unknown): value is string {
  return typeof value === "string";
}
```

👉 핵심:

* unknown 처리할 때 필수
* 안전한 코드의 핵심

---

# 🔥 5. 제네릭으로 재사용 함수 만들기

공통 로직에 사용

```ts
function getFirst<T>(arr: T[]): T {
  return arr[0];
}
```

👉 핵심:

* 유틸 함수, 훅, 라이브러리에서 많이 씀

---

# 🔥 6. Record 패턴 (객체 맵)

키-값 구조 깔끔하게 정의

```ts
type UserRole = "admin" | "user";

const roleMap: Record<UserRole, string> = {
  admin: "관리자",
  user: "일반유저",
};
```

👉 핵심:

* enum보다 가볍고 실무에서 더 많이 씀

---

# 🔥 7. 함수 타입 명확하게 정의

특히 콜백에서 중요

```ts
type Add = (a: number, b: number) => number;

const add: Add = (a, b) => a + b;
```

👉 핵심:

* 협업 시 함수 계약을 명확하게

# 🔥 8. 옵셔널 & 기본값 패턴

```ts
type Props = {
  name?: string;
};

function greet({ name = "익명" }: Props) {
  console.log(name);
}
```

👉 핵심:

* React에서도 매우 자주 사용

# 🔥 9. as const (리터럴 고정)

const status = ["loading", "success", "error"] as const;

👉 핵심:

* 문자열을 정확한 타입으로 유지

# 🔥 10. 에러 방지 패턴 (never 체크)


# 🔥 1. “분산 시스템 기본기” (프론트도 필수)

타입 잘 써도 여기 모르면 한계 온다.

### 👉 반드시 이해

* consistency vs availability (CAP)
* eventual consistency
* idempotency
* retry / backoff

👉 왜 중요하냐면

* API 설계
* 상태 관리
* 캐싱 전략

전부 여기에 걸린다

👉 핵심
**“서버는 항상 깨진다”를 전제로 설계**

---

# 🔥 2. 캐싱 전략 + 데이터 동기화

React Query 쓰는 수준에서 끝내면 안 된다

### 👉 깊게 들어갈 것

* stale-while-revalidate
* optimistic update
* cache invalidation 전략

관련 참고:

* TanStack Query

👉 핵심
**“데이터는 언제나 낡아 있다”**

---

# 🔥 3. 네트워크 & 브라우저 내부

여기 모르면 퍼포먼스 못 잡는다

### 👉 필수 영역

* HTTP/1.1 vs HTTP/2 vs HTTP/3
* 브라우저 렌더링 파이프라인
* event loop

👉 핵심
**“코드 실행 환경을 이해해야 최적화가 가능”**

---

# 🔥 4. 렌더링 전략 (Next.js 이상 레벨)

SSR/CSR 아는 수준은 부족

### 👉 들어갈 것

* streaming SSR
* partial hydration
* server components

관련:

* Next.js

👉 핵심
**“언제, 어디서 렌더링할지 결정하는 능력”**

# 🔥 5. 테스트 전략 (진짜 실무형)

단위 테스트만 하면 반쪽짜리다

### 👉 확장

* integration test
* e2e test
* contract test

도구:

* Playwright
* Jest

👉 핵심
**“버그를 잡는 게 아니라 ‘깨지지 않게 만든다’”**

# 🔥 6. CI/CD + 품질 자동화

여기 못 하면 팀 생산성 못 올린다

### 👉 배울 것

* lint / type / test 파이프라인
* preview deployment
* canary release

👉 핵심
**“코드 품질은 사람이 아니라 시스템이 책임진다”**

도구 예:

👉 핵심

👉 핵심

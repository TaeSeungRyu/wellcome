## 📊 Fiber 아키텍처 기반 Reconciliation 설명

### 1️⃣ 왜 Fiber가 등장했는가?

기존 **Stack Reconciler** 방식은 한 번 렌더링이 시작되면
끝날 때까지 **중단(interrupt)할 수 없었습니다.**

→ 대규모 트리에서는 메인 스레드를 오래 점유
→ 애니메이션/입력 지연 발생

이를 해결하기 위해 **React**는
**Fiber 아키텍처 (React 16+)**를 도입했습니다.

---

## 2️⃣ Fiber란?

Fiber는 React 내부에서 사용하는 **작업 단위(Unit of Work)**입니다.

기존:

```
재귀 호출 기반 (Call Stack 의존)
→ 중단 불가
```

Fiber:

```
Linked List 구조의 노드 단위 처리
→ 작업을 쪼개서 실행 가능
→ 중단 / 재개 가능
```

즉,

> Reconciliation을 "조각내서" 수행할 수 있게 만든 구조

---

## 3️⃣ Fiber 기반 Reconciliation 과정

### 🔹 1단계: Render Phase (비동기 가능)

* 새로운 Virtual DOM 생성
* 이전 Fiber와 비교 (Diff)
* 변경 사항 계산
* **중간에 멈출 수 있음**

이 단계는:

* interrupt 가능
* 우선순위 조정 가능

---

### 🔹 2단계: Commit Phase (동기, 중단 불가)

* 실제 DOM 변경
* useEffect 실행
* ref 업데이트

👉 이 단계는 매우 빠르게 수행됨

---

## 4️⃣ Fiber의 핵심 개념

### ✅ 1. 작업 분할 (Time Slicing)

작업을 잘게 나눠 브라우저에 제어권을 돌려줌

### ✅ 2. 우선순위 기반 처리 (Scheduling)

* 사용자 입력 → 높은 우선순위
* 데이터 로딩 → 낮은 우선순위

### ✅ 3. Double Buffering

* current tree
* workInProgress tree

완성 후 교체

---

## 🧠 Concurrent Mode와의 관계

### 1️⃣ Concurrent Mode란?

React가 렌더링을 **중단, 재시작, 우선순위 조정**할 수 있게 하는 실행 모드

👉 이 기능이 가능한 이유가 바로 **Fiber 구조**

---

## 2️⃣ 핵심 연결 구조

| 개념              | 역할               |
| --------------- | ---------------- |
| Fiber           | 작업을 쪼갤 수 있는 구조   |
| Concurrent Mode | 그 구조를 활용하는 실행 전략 |

즉,

> Fiber는 구조
> Concurrent는 동작 방식

---

## 3️⃣ Concurrent 기능 예시

### 🔹 startTransition

긴급하지 않은 업데이트를 낮은 우선순위로 처리

```jsx id="2bkl7s"
startTransition(() => {
  setList(filteredData);
});
```

### 🔹 Suspense

비동기 로딩을 자연스럽게 처리

---

## 4️⃣ 기존 동기 렌더링 vs Concurrent

| 구분     | 기존    | Concurrent |
| ------ | ----- | ---------- |
| 렌더 중단  | ❌ 불가  | ✅ 가능       |
| 우선순위   | 없음    | 있음         |
| 사용자 경험 | 지연 가능 | 부드러움       |

---

## 5️⃣ 면접용 핵심 요약

* Fiber는 React의 새로운 Reconciliation 구조
* 작업을 잘게 나눠 중단/재개 가능
* Render Phase는 interrupt 가능
* Commit Phase는 동기 실행
* Concurrent Mode는 Fiber 기반으로 동작
* 결과적으로 사용자 경험(UX)을 개선

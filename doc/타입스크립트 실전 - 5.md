

# 🔥 1. Schema → Type → Runtime 통합 설계


* Zod를 “검증용”이 아니라 **싱글 소스 오브 트루스**로 사용
* API / FE / 테스트 전부 같은 스키마 공유

```ts
const UserSchema = z.object({
  id: z.number(),
  name: z.string(),
});

type User = z.infer<typeof UserSchema>;
```

### 👉 실전 확장

* OpenAPI ↔ Zod 자동 생성
* tRPC 구조 이해

👉 핵심
**“타입 따로, 검증 따로” → ❌
“스키마 하나로 통합” → ✅**

# 🔥 4. “타입 안전 API 클라이언트” 만들기

이건 실무 체감 가장 큼.

### 👉 목표

* fetch 래퍼 만들기
* 요청/응답 타입 강제

```ts
async function request<T>(
  url: string,
  schema: z.ZodSchema<T>
): Promise<T> {
  const res = await fetch(url);
  const data = await res.json();
  return schema.parse(data);
}
```

👉 확장

* 에러 타입까지 포함
* retry / timeout / cancel

👉 핵심
**API 호출 = 타입 계약 실행**

# 🔥 5. “브랜드 타입 + 도메인 모델링” 심화

지금 쓴 건 입문 수준이다.

### 👉 다음 단계

* 값 객체(Value Object) 패턴
* 불변 도메인 모델

```ts
type Email = string & { readonly brand: "Email" };

function createEmail(value: string): Email {
  if (!value.includes("@")) throw new Error();
  return value as Email;
}
```

👉 핵심
**Primitive → Domain Type으로 승격**

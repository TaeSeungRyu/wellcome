> **React와 Vue.js에서도 임계 렌더링 경로(Critical Rendering Path)는 그대로 존재합니다.**
> 다만 **동작 방식이 약간 다릅니다.**

---

## ✅ 1️⃣ 기본 원리: 동일하게 존재

브라우저가 렌더링하는 이상 CRP는 항상 존재합니다.

흐름:

```
HTML → DOM
CSS → CSSOM
JS 실행
Render Tree → Layout → Paint
```

즉,

- React든
- Vue든
- 순수 HTML이든

**브라우저 렌더링 구조는 동일합니다.**

---

## ✅ 2️⃣ SPA에서는 특징이 있음

React / Vue는 보통 SPA 구조입니다.

### 일반 HTML

```html
<h1>Hello</h1>
```

→ 바로 화면 표시 가능

---

### React / Vue

```html
<div id="app"></div>
<script src="bundle.js"></script>
```

동작:

1. HTML 다운로드
2. JS 다운로드
3. JS 실행
4. Virtual DOM 생성
5. DOM 생성
6. 렌더링

즉:

> JS가 끝나야 화면이 나옵니다.

그래서 SPA는:

- 초기 렌더링 느릴 수 있음

---

## ✅ 3️⃣ SPA의 Critical Rendering Path

### 순수 HTML

```
HTML → DOM → Paint
```

빠름

---

### React / Vue SPA

```
HTML
↓
JS 다운로드
↓
JS 실행
↓
DOM 생성
↓
Paint
```

더 길어짐

---

## ✅ 4️⃣ 그래서 SSR이 존재

이 문제 때문에 SSR이 등장했습니다.

예:

- React → Next.js
- Vue → Nuxt

SSR:

```
서버에서 HTML 생성
↓
브라우저 표시
↓
JS hydrate
```

즉:

> CRP가 짧아짐

---

## ✅ 5️⃣ 핵심 차이

| 구분        | 순수 HTML | React/Vue SPA |
| ----------- | --------- | ------------- |
| HTML        | 완성됨    | 비어있음      |
| JS 의존성   | 낮음      | 높음          |
| 초기 렌더링 | 빠름      | 느릴 수 있음  |
| CRP 길이    | 짧음      | 김            |

---

## ✅ 6️⃣ 중요한 개념

### ✔ SPA

> JS가 Critical Rendering Path에 포함됨

### ✔ SSR

> HTML이 먼저 렌더링됨

---

## ✅ 핵심 정리

### ✔ Vue / React

- 임계 렌더링 경로 존재
- JS 영향 큼

### ✔ SPA

- JS가 끝나야 렌더링

### ✔ SSR

- HTML 먼저 렌더링

---

## ✅ 한 줄 정리

> **React/Vue는 CRP가 더 길어지기 쉽고, SSR은 CRP를 줄이기 위한 기술이다.**
